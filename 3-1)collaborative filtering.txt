## 추천 시스템의 기본인 협업 필터링 (Collaborative Filtering)은 크게 user-based filtering, item-based filtering 방식으로 나뉘어져 있다. 각각에 대해서 이해하고,  
2-2에서 구해 본 응답자의 “스타일 선호 정보표”를 토대로 Validation 데이터 내 응답자의 “스타일 선호 여부 예측” 문제를 2가지 기법으로 어떻게 적용해 볼 수 있고, 서로 비교하여 
어떤 장단점을 갖는지 설명

# User-based Collaborative Filtering

- **아이디어**: 비슷한 취향을 가진 사용자들이 유사한 항목을 좋아할 것이라는 가정에 기반, 대량의 데이터를 처리할 때 더 우수한 성능을 발휘
- **방법**: 각 사용자가 평가한 항목(영화, 책 등)에 대한 정보를 바탕으로, 다른 사용자가 좋아할 만한 항목을 추천, 피어슨 상관계수를 일반적으로 이용

**유저 기반 협업 필터링 User-based Collaborative Filtering**

1/ 유저를 n개의 아이템으로 구성된 벡터로 표현

2/ 피어슨 상관계수를 활용하여 타겟 유저와 가장 유사한 k명의 유저를 파악

3/ k명의 유저가 각 아이템에 준 평점을 평균하여 타겟 유저의 평점을 예측하고, 예측 평점이 높은 순으로 추천

- 장점: 정확한 추천 가능
- **단점**: 사용자가 많아지면 유사한 사용자를 찾는 데 시간이 많이 걸리고, 데이터가 희소할 때(모든 사용자가 모든 항목을 평가하지 않는 경우) 성능이 떨어짐, 확장성 부족>ICBF개발, cold-start 문제

# Item-based Collaborative Filtering

- **아이디어**: 비슷한 항목들은 동일한 사용자들에 의해 비슷하게 평가될 것이라는 가정에 기반, 데이터셋이 작은 경우 더 좋은 성능, 아이템이 많지만 평점이 상대적으로 적은 추천시 스템에서도 잘 작동. 또한 아이템 간의 관계가 비교적 정적이기 때문에 더 적은 온라인 계산으로 UBCF과 동일한 품질을 제공
- **방법**: 항목들 간의 유사성을 계산하여, 사용자가 과거에 좋아한 항목과 유사한 다른 항목을 추천

**아이템 기반 협업 필터링 Item-based Collaborative Filtering**

1/ 아이템을 n명의 유저로 구성된 벡터로 표현

2/ 코사인 유사도를 활용하여 다른 아이템과의 유사성 확인

3/ 유사한 아이템들의 평점을 가중 평균하여 (코사인 유사도를 가중치로 사용) 평점을 예측하고, 예측값이 높은 순으로 추천

- **장점**: 항목 간의 유사성을 미리 계산해두면, 새로운 사용자가 들어왔을 때 빠르게 추천이 가능,사용자의 관심을 모델링하고 온라인 개인화를 쉽게 구현할 수 있다는 강점
- 단점: Cold-start 문제 극복 못함, 아이템이 많고, 아이템에 대한 사용자들의 선호도 데이터가 적을 때, 추천에 대한 성능이 제한> data sparsity 문제

# Collaborative Filtering의 장단점

- 방법: 메모리 기반 협업 필터링
- **장점**: Collaborative Filtering은 사용자들이 직접 평가한 데이터에만 의존하므로, 사용자가 표현하지 않은 명시적인 취향(성별, 나이, 선호 장르 등)에 구애받지 않고 유연하게 작동
- **단점**: 데이터가 충분하지 않은 경우(새로운 사용자 또는 항목이 많은 경우) 성능이 떨어지는 "Cold Start Problem"이 발생할 수 있습니다. 또한 사용자가 평가한 데이터가 적을 때는 추천 정확도가 낮아질 수 있음
